# Backlog 模板

PRD から降格した需求や、開発中に発見した改善点の待評価プール。

## 模板

```markdown
# {项目名} Backlog

> 开发过程中发现的改善点和新功能想法。经评估后可纳入 PRD。

## {Section 名}

| #    | 状态 | 优先级 | 内容 | 难度 | 说明 |
| ---- | ---- | ------ | ---- | ---- | ---- |
| #xxx | ❌   | P1     | xxx  | 🟡   | xxx  |

## Feature Ideas

| #    | 状态 | 优先级 | 内容 | 难度 | 说明 |
| ---- | ---- | ------ | ---- | ---- | ---- |
| #xxx | ❌   | —      | xxx  | 🟡   | xxx  |

> 难度：🟢 半小时内 / 🟡 半天 / 🔴 1天+
> 优先级：P0 最优先 / P1 近期 / P2 后续 / — 未定
```

## 写法要点

- Section 按页面或模块分组，Feature Ideas 放通用想法
- **编号延续 PRD/archive 的全局编号**，不重新从 1 开始
- 同一 section 内编号可能不连续（全局递增，跨 section 分配）
- 难度用 🟢🟡🔴 直观标识
- 优先级提升到 P0/P1 时，将该项移入 PRD 当前阶段（保留原编号）
- 可用 `/add-backlog` command 快速添加
